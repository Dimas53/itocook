## Идея и проблема

В офисе коллеги часто готовят обеды по очереди или заказывают еду вскладчину. Сейчас это решается через мессенджеры, таблицы в Excel и устные договорённости: кто платил, кто должен, кто готовит сегодня.

**ItoCook** — внутреннее веб-приложение, которое автоматизирует весь этот процесс: координация обедов, учёт расходов, баланс каждого участника и уведомления — всё в одном месте.

💡 **Важно:** **ItoCook** — это не просто приложение для готовки. По своей сути это **Splitwise для рабочих процессов**: общий котёл, учёт расходов и координация участников. Платформу можно масштабировать под любые нужды компании:

| Сценарий | Пример |
|---|---|
| 🍽️ Офисные обеды | основной сценарий |
| ✈️ Командировки | делим расходы на гостиницу, бензин, питание |
| 🎉 Корпоративы | общий сбор, учёт кто сколько внёс |
| 🛒 Закупки для офиса | кофе, канцелярия, хозтовары |

## Функции

**Для всех**

- Главная: кто готовит, что, статус обеда, текущий баланс
- «Я обедаю» — запись в один клик, отказ за 24 ч до
- Меню на неделю — резервирование, уведомления за 30 ч, согласие за 24 ч до
- Очередь поваров — свободный календарь на неделю/месяц
- Календарь уборки кухни — просмотр для всех, редактирование для админа
- Анонимные оценки и отзывы по блюду
- Голосование: лучшее блюдо / повар недели

**Для повара**

- «Я готовлю сегодня» — уведомление всем, обновление стейта
- Выбор блюда из истории или ввод вручную
- Загрузка чека — фото или сумма вручную, автоматическое деление между участниками
- Рецепты: добавление, фото, ингредиенты, шаги
- Пересчёт ингредиентов при изменении числа участников
- Список покупок — из рецепта или вручную, экспорт текстом

**Уведомления**

- Напоминание с 8:00 если повар не назначен (повтор каждые 30 мин до 10:00)
- Напоминание о дежурстве на кухне — за 12 ч и в 9:00, кнопка подтверждения
- Красное уведомление при минусовом балансе (< −10 €)
- Push / Email / WhatsApp — каждый настраивает в профиле

**Финансы** *(только Admin / Бухгалтер)*

- Все балансы сотрудников, пополнение баланса и общий остаток
- История расходов, ручное добавление закупок

## MVP — минимальный рабочий сценарий

Один рабочий день полностью в приложении:

```
Утро:
  [Повар] → нажимает "Я готовлю" → уведомление всем
  [Повар] → выбирает блюдо из истории или вводит вручную
 
До обеда:
  [Участник] → видит на главной: кто готовит, что, когда
  [Участник] → нажимает "Я обедаю" → попадает в список
  [Участник] → видит свой баланс рядом с кнопкой
 
После готовки:
  [Повар] → нажимает "Обед готов" → уведомление участникам
  [Повар] → вводит сумму чека (напр. 47,80 €)
  [Система] → делит на 8 участников → списывает по 5,98 € с каждого
 
Итог:
  [Участник] → видит обновлённый баланс
  [Бухгалтер] → видит транзакцию в финансовом разделе
```

## AI-ассистент

Встроенный кулинарный чат через **OpenRouter API** — не нужен собственный GPU.


Модель: `google/gemini-2.0-flash-lite` — быстро, дёшево,
достаточно для структурированных JSON-ответов.
Через OpenRouter можно переключить на любую другую модель
без изменения кода.

Принцип: **отвечает только на кулинарные вопросы**. На всё остальное — вежливый отказ. Ограничение реализуется через системный промпт. Ничего не сохраняется автоматически, пользователь сам решает что добавить.

**Три сценария:**

1. *«Придумай суп на 12 человек»* → карточка с ингредиентами и шагами → «Добавить в рецепты» / «В список покупок»
2. Открываешь рецепт на 5 человек, нажимаешь «Пересчитать» → вводишь 15 → новые пропорции
3. *«Чем заменить сметану?»* → предложение → «Заменить в рецепте»


Формат ответа ассистента (JSON):

```json
{
  "title": "Куриный суп",
  "servings": 12,
  "time_minutes": 90,
  "ingredients": [
    { "name": "Курица", "amount": 2.4, "unit": "кг" },
    { "name": "Картофель", "amount": 3.0, "unit": "кг" }
  ],
  "steps": ["Залить курицу холодной водой...", "Варить 40 минут..."],
  "message": "Рецепт рассчитан на 12 порций."
}
```

Фронтенд парсит JSON и рендерит красивую карточку — не просто текст в чате.

### Шаг 2 — Docker: запускаем бэкенд без написания кода
Пишем один файл `docker-compose.yml` — и Directus с PostgreSQL
уже работают локально. Код ещё не написан, но база и API готовы.

- [ ] Написать `docker-compose.yml`
- [ ] Создать `.env` с паролями
- [ ] Запустить: `docker-compose up -d`
- [ ] Проверка: открыть `localhost:8055` → экран логина Directus

```yaml
services:
  database:
    image: postgres:15
    environment:
      POSTGRES_USER: itouser
      POSTGRES_PASSWORD: itopassword
      POSTGRES_DB: itocook_db

  directus:
    image: directus/directus:latest
    ports:
      - "8055:8055"
    depends_on:
      - database
    environment:
      DB_CLIENT: "pg"
      DB_HOST: "database"
      DB_DATABASE: "itocook_db"
      ADMIN_EMAIL: "admin@itocook.com"
      ADMIN_PASSWORD: "admin"
```

✅ *После этого шага: база данных и API работают, код ещё не написан.*

- [ ] Эндпоинт расчёта суммы на участника
