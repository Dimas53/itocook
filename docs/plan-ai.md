# ItoCook — Проектная заметка

---

## 1. Идея проекта

Внутреннее веб-приложение для координации офисных обедов. Коллеги готовят по очереди, скидываются на продукты, ведут баланс. Всё в одном месте: кто готовит, что готовит, сколько стоит, кто участвует.

**Варианты названия:** ItoCook / ItoChef / ItoMeal / ItoTasty / Cucito

---

## 4. MVP — один рабочий день в приложении

### Утро
- Сотрудник заходит, нажимает **"Я готовлю сегодня"**
- Всем уходит email: *"Сегодня готовит [имя], блюдо будет объявлено позже"*
- Повар выбирает блюдо из истории или вводит название вручную

### До обеда
- Каждый видит на главной: кто готовит и что
- Нажимает **"Я обедаю"** → попадает в список участников
- Видит свой текущий баланс рядом с кнопкой

### После готовки
- Повар нажимает **"Обед готов"** → все участники получают email
- Повар вводит сумму чека (фото — на потом)
- Приложение делит сумму на количество участников и списывает с каждого

### Итог дня
- Каждый видит обновлённый баланс
- Бухгалтер видит всю транзакцию в финансовой странице

## 5. Полный список функций

### 5.1 Ключевые функции

- **"Я повар сегодня"** — назначение себя в свободной форме, если никто не в очереди. Уведомление всем. Обновление стейта.
- **"Согласен на обед"** — кнопка для участников. Переход из уведомления (email/WhatsApp) → приложение → клик. Связь с финансовым расчётом.
- **"Обед готов"** — кнопка для повара. Уведомление всем участникам.
- **Логин** — персональный для сотрудников, общий для бухгалтера/админа.
- **Календарь уборки кухни** — ручное или автоматическое назначение по отделам и сотрудникам. Отображение на месяц/неделю.

### 5.2 Функции для повара

- Добавление фото чека и общей стоимости (обязательно — для расчёта)
- История блюд — список с поиском по названиям
- Добавление фото и рецепта блюда — сохранение в базу, редактирование, поля для фото и продуктов
- Выбор блюда из истории для сегодняшнего дня
- Пересчёт количества ингредиентов при изменении числа участников
- Добавление продуктов в базу для списка покупок
- Формирование списка покупок — из рецептов / базы / вручную. Экспорт в текстовый формат.

### 5.3 Функции для всех

- Быстрая запись на обед с главной в один клик (и отказ за 24 ч до)
- Отображение текущего баланса при записи
- Блоки на главной: кто готовит, блюдо, статус обеда, календарь уборки
- Очередь поваров — свободный календарь для желающих готовить
- Меню на неделю — резервирование места повара, уведомления, ограничения на согласие (максимум за 24 ч до, повтор за 30 ч до)

### 5.4 Интерактив и голосования

- Анонимные оценки и отзывы по блюду/повару
- Голосование раз в неделю: лучшее блюдо недели, повар недели (опционально)
- Интеграция 1–2 API книг рецептов (опционально)

### 5.5 Финансы и баланс

- Расчёт стоимости — после загрузки чека/суммы, деление между участниками
- Баланс пользователей — ввод суммы вручную (только кэш через бухгалтерию)
- Общий баланс — все счета и общий остаток (только для админа/бухгалтера)
- Общий расход за период — все расходы + ручное добавление закупок (опционально)

### 5.6 Уведомления

- Напоминание о записи на обед всем (Push / Email / WhatsApp)
- Напоминание о готовности обеда участникам
- Напоминание с 8:00 до 10:00 (повтор), если повар не назначен
- Уведомление о минусовом балансе (если < −10–20 €) — красный цвет, частые напоминания
- Напоминание о дежурстве на кухне — за 12 ч до и в 9:00, кнопка подтверждения
- Настройка типа уведомлений в профиле (Push / Email / WhatsApp)
- Аналитика в админке: кто не подтверждает участие

- Календарь уборки кухни

**Финансы**
- История расходов (чеки, суммы за обеды)

**Интерактив**
- Оценки и отзывы
- Голосования

**Уведомления**
- Триггеры по статусам (обед готов, запись, минусовой баланс)
- Связь с email / push

## 10. AI-ассистент (кулинарный чат)

### Идея

Встроенный чат внутри ItoCook, специализированный **только на кулинарии**. Работает через OpenRouter API. Пользователь не покидает приложение.

### Ключевые принципы

- Отвечает только на кулинарные вопросы (рецепты, ингредиенты, порции, техники)
- На посторонние темы вежливо отказывает
- Ограничение — через системный промпт
- Ничего не сохраняется автоматически — пользователь решает сам

### Технический стек

| Параметр | Значение |
|----------|----------|
| API | OpenRouter (`/api/v1/chat/completions`) |
| Модель | `google/gemini-2.0-flash-lite` |
| Формат ответа | JSON |
| История чата | Только в рамках сессии |
| Автосохранение | Нет — только по явному действию |

### Системный промпт

```
Ты кулинарный помощник приложения ItoCook.
Отвечай только на вопросы о рецептах, ингредиентах, порциях,
времени и технике приготовления.
На вопросы не по теме вежливо отвечай, что можешь помочь
только с кулинарией.
Всегда возвращай ответ в формате JSON (см. схему ниже).
```

### Формат ответа (JSON)

```json
{
  "title": "Куриный суп",
  "servings": 12,
  "time_minutes": 90,
  "ingredients": [
    { "name": "Курица", "amount": 2.4, "unit": "кг" },
    { "name": "Картофель", "amount": 3.0, "unit": "кг" },
    { "name": "Морковь", "amount": 3, "unit": "шт" },
    { "name": "Лук репчатый", "amount": 2, "unit": "шт" },
    { "name": "Соль", "amount": 2, "unit": "ч.л." }
  ],
  "steps": [
    "Залить курицу холодной водой, довести до кипения, снять пену.",
    "Варить бульон 40 минут на среднем огне.",
    "Добавить крупно нарезанные картофель, морковь и лук.",
    "Варить ещё 25 минут до готовности овощей.",
    "Посолить по вкусу, подавать горячим."
  ],
  "message": "Рецепт рассчитан на 12 порций. Время варки может немного варьироваться в зависимости от размера кусков."
}
```

### Сценарии использования

**Сценарий 1 — Новый рецепт через чат**
1. Пользователь пишет: *«Придумай суп из курицы на 12 человек»*
2. Ассистент возвращает JSON с рецептом
3. Фронтенд рендерит карточку с ингредиентами и шагами
4. Кнопки:
    - **«Добавить в рецепты»** → черновик рецепта, можно отредактировать и сохранить
    - **«Добавить ингредиенты в список покупок»** → летят в закупки на сегодня/неделю

**Сценарий 2 — Пересчёт существующего рецепта**
1. Пользователь открывает рецепт (например, на 5 человек)
2. Нажимает **«Пересчитать»**, вводит нужное количество (например, 15)
3. Рецепт + новое количество передаются ассистенту как контекст
4. Ассистент возвращает пересчитанный JSON
5. Кнопка **«Сохранить как новый рецепт»**

**Сценарий 3 — Замена ингредиента**
1. Пользователь: *«Чем можно заменить сметану в этом рецепте?»*
2. Кнопка **«Заменить в рецепте»** → подмена ингредиента, предложение сохранить

### Контекст, передаваемый ассистенту

- Текущий рецепт (название, ингредиенты, шаги, количество порций)
- Запрошенное количество порций
- Список блюд на неделю (опционально)
